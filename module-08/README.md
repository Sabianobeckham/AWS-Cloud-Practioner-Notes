AWS Cloud Practitioner Learning Journey – Module 08: AI, Machine Learning & Data Analytics

This week I explored how AWS supports Artificial Intelligence, Machine Learning, Generative AI, and Big Data Analytics.
Some of the key concepts I learned include:
🔹 The difference between Artificial Intelligence, Machine Learning, Deep Learning, and Generative AI.
🔹 How the ETL (Extract, Transform, Load) process prepares data for analytics and machine learning.
🔹 Building data lakes with Amazon S3 and analyzing data using services such as Amazon Redshift, AWS Glue, Amazon EMR, Amazon Athena, Amazon Kinesis, and Amazon QuickSight.
🔹 AWS AI services including Amazon Comprehend, Textract, Rekognition, Translate, Lex, Personalize, and Kendra.
🔹 Enterprise Generative AI services such as Amazon Bedrock, Amazon Q Business, Amazon Q Developer, and Amazon SageMaker for building and deploying intelligent applications.
One lesson that stood out to me is this:
AI is only as effective as the quality of the data behind it. Clean, well-structured, and reliable data is the foundation of every successful AI and analytics solution.
